BEAM NAVIGATION REDESIGN

Navigation redesign for Beam, a Nutanix Cloud Cost Governance and Security Compliance Management platform.

OVERVIEW

This project was done for Nutanix, a leading Hyperconverged Infrastructure company which builds software and hardware for enterprise IT admins, to help them manage and store data in their data centers or on the cloud.

Nutanix acquired Botmetric, a cloud Cost Governance and Security Compliance management platform in Feb 2018, post which it was renamed Beam and went a complete visual and navigational overhaul. This project mainly focuses on the process of information architechture and navigation redesign for Beam.

ROLE AND DURATION

UX research, information architechture, interaction design, visual design

Design lead, 2 product managers and many developers

6 months (Jul to Dec 2018)

WHAT DOES BEAM DO ?

Beam is a multi-cloud optimization service delivered as part of the Nutanix Enterprise Cloud OS. It provides organizations with deep visibility and rich analytics detailing cloud consumption patterns, along with one-click cost optimization across their cloud environments. It also provides insights into cloud compliance and security vulnerabilities in real-time, thus, helping to resolve potential threats.

THE REDESIGN PROCESS

The redesign process was an interative journey. It included, first, understanding the product and the users, how they use the product, and the problems they have, followed by mapping the current information architechture and the problems with the current navigation. It was critical to get all these insights before embarking on the redesign itself to ensure that the right problems were getting solved in alignment with the project goals.

To understand the pain points, we had multiple sessions with the internal beam team and went through a lot of customer feedback. Along with that I also studied the existing product thoroughly to identify design issues and challenges with the existing information architechture and navigation.

SOLVING THE PROBLEMS

PROBLEM 1 : Users could not locate where the reports section is. The report icon is not intuitive.
PROBLEM 2 : Selecting the cloud and account is a main action in Beam. Right now changing it on the right changes the options on the left.

PROBLEM 3 : Cost center view does not exist in the current UI.
PROBLEM 4 : The cloud and account view switcher does not have an all payer account view.
PROBLEM 5 : AWS can have multiple payer accounts which can in turn have multiple linked accounts upto 100. The current collapsible scroll option has discoverability issues.
PROBLEM 6 : Search is not global. It is only within each cloud.
PROBLEM 7 : Users cannot bookmark a particular view if they want to go back to it again and again.

PROBLEM 7 : Many of the internal pages had 4 levels of horizontal tabs which gave less space for the actual content.
PROBLEM 8 : Lack of grouping

USER TESTING

The iterations and the new design were first reviewed multiple times internally by the design team and the stakeholders. Next it was user tested with existing Beam customers. It was also user tested with 5 users at the NuExperience Lab in Mumbai, where we conducted usability testing sessions by giving them tasks to perform and their response was recorded.

The feedback from the usability testing sessions were further incorporated in the new navigation redesign before handoff to developers.

FURTHER STEPS

Currently, these designs are in development and should be in the market soon. The next steps would be to gather user feedback once the product is live and work with the help of those to improve the product further.

KEY TAKEAWAYS

1.  Working on a fast paced live project taught me to iterate, discard, iterate and so on, and quickly incorporate feedback and changes to meet the deliverables.

2.  Another key learning was how to convince the stakeholders to quickly reach a design decision. Whenever there is lack of clarity, quick feedback from a few existing users really helps.

3.  As the lead designer of the project I had to be aggressive in constantly asking what is happening next and pushing for both reviews and development.
